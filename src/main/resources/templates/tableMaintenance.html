Understanding Table Maintenance
Maintenance Actions
Vacuum
Reclaims storage from deleted/updated rows. Non-blocking.
Analyse
Updates statistics for query planner. Non-blocking.
Vacuum Full
Compacts table. Requires exclusive lock. Use for high bloat.
Quick Commands
-- Vacuum a specific table
VACUUM VERBOSE schema.tablename;

-- Analyse a specific table
ANALYSE VERBOSE schema.tablename;

-- Vacuum and analyse together
VACUUM ANALYSE schema.tablename;

-- Vacuum full (requires downtime)
VACUUM FULL VERBOSE schema.tablename;

Note: PostgreSQL's autovacuum should handle most maintenance automatically. If you see many recommendations here, consider tuning autovacuum parameters. Dead tuples are normal during updates/deletes but should not accumulate excessively.
